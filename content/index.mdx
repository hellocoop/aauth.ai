  </h1>
  <p className="text-xl text-gray-600 dark:text-gray-400 max-w-3xl mx-auto mb-4" style={{lineHeight: '1.6'}}>
    AAuth is a protocol family for agent identity, resource access, and user delegation — designed for open ecosystems where pre-registration, browser redirects, and bearer tokens don't fit.
  </p>
  <p className="text-lg text-gray-500 dark:text-gray-500 max-w-2xl mx-auto mb-10" style={{lineHeight: '1.6'}}>
    Every agent gets its own cryptographic identity. A domain, static metadata, and a JWKS — no portal, no bilateral agreement, no shared secret. This is the foundation that authorization, governance, and federation build on.
  </p>
  <div className="flex justify-center gap-4 flex-wrap">

<div className="mb-20">
  <h2 className="text-2xl font-bold text-center mb-4">Agents Are Different</h2>
  <p className="text-center text-gray-600 dark:text-gray-400 max-w-2xl mx-auto mb-12">
    Agents discover resources at runtime. They execute long-running tasks across trust domains. They need authorization mid-task and may need to explain themselves. A protocol designed for pre-registered clients with fixed integrations cannot serve agents that discover their needs as they go.
  </p>
  <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">No Pre-Registration</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Agents self-publish identity at HTTPS URLs. Any agent can interact with any resource on first contact — no admin portal, no client secrets.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Per-Instance Identity</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Each agent instance gets its own identifier and signing key, enabling granular revocation and audit — no more shared client secrets.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Proof of Possession</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Every request is signed with HTTP Message Signatures. Every token is key-bound. Stolen tokens are worthless without the signing key.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Async by Design</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        One pattern — <code>202 Accepted</code> + polling — handles immediate grants, user consent, headless agents, enterprise approval, and clarification chat.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Progressive Trust</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Resources declare the trust level they need: pseudonymous, identified, or fully authorized. Escalate within the same protocol.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Tool-Call Governance</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Agent and Person Server work together directly — the PS manages what tools the agent can call, providing permission and audit for tool use.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Resource Identity</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        Resources sign their own tokens, binding access challenges to their cryptographic identity. Prevents confused deputy attacks at the protocol level.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Clarification Chat</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        During consent, users can ask agents questions about their intent. Agents can explain or adjust requests — enabling informed consent beyond static scopes.
      </p>
    </div>
    <div className="p-6 rounded-xl border border-gray-200 dark:border-gray-700/50 bg-white dark:bg-gray-900/50">
      <h3 className="text-lg font-semibold mb-2">Cross-Domain Federation</h3>
      <p className="text-sm text-gray-600 dark:text-gray-400 leading-relaxed">
        PS-to-AS federation enables access across trust domains without the agent needing to know about each access server. Call chaining passes auth downstream.
      </p>
    </div>
  </div>
</div>
